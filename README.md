# Kebir Mermercilik — نسخة Static (بدون PHP)

## شو تغيّر
- كل صفحات `.php` صارت `.html` عادي.
- تسجيل الدخول/التسجيل والـ "session" يلي كانوا بـ PHP (`session_start`, `$_SESSION`) صاروا يشتغلوا بجافاسكريبت + `localStorage` بالمتصفح (ملف `js/auth.js`).
- المستخدمين القدام (بملف `users.json`) محفوظين متل ما هنن بالضبط، بكلمات المرور المشفّرة (bcrypt) — استخدمت مكتبة `bcryptjs` بالمتصفح عشان تتحقق من نفس الـ hash القديم بدون ما احتاج أعرف كلمات المرور الأصلية. يعني تقدر تسجلي دخول بنفس الحسابات القديمة وكلمات مرورها الأصلية.
- أي حساب جديد بينسجل من صفحة `register.html` بينحفظ محليًا بمتصفح المستخدم (localStorage)، لأنه ما في سيرفر أو قاعدة بيانات حقيقية تخزن الداتا بشكل دائم ومشترك بين الأجهزة.
- صفحة `tasurunleri.php` (يلي كانت تاخد `tur_id` من الرابط) صارت `tasurunleri.html?tur_id=1` وبتشتغل بجافاسكريبت.

## قيود مهمة (لازم تعرفيها)
1. **ما في قاعدة بيانات حقيقية.** بما إنه الموقع Static 100%، أي بيانات تنحفظ (حسابات جديدة) بتضل محفوظة بس بمتصفح نفس الجهاز يلي سجل فيه المستخدم. ما رح تنشارك بين الأجهزة أو تنحفظ عالسيرفر.
2. **الصور الخارجية** (يلي كانت روابط Bing/مواقع تانية بالكود الأصلي) خليتها متل ما هي، لأنها روابط خارجية مو ملفات بالمشروع.
3. لتشغيل الموقع، لازم تفتحيه عن طريق سيرفر محلي بسيط (مو بس فتح الملف مباشرة بالمتصفح)، لأنه بيستخدم `fetch()` لتحميل `users.json`. مثلاً:
   ```bash
   cd my_project_static
   python3 -m http.server 8000
   ```
   وبعدها افتحي `http://localhost:8000` بالمتصفح.

## لو بدك سيرفر حقيقي بالمستقبل
إذا حبيتي لاحقًا تخزين حقيقي (قاعدة بيانات، حسابات مشتركة بين الأجهزة)، لازم Backend فعلي (Node.js/Express, Python/Flask, إلخ) بدل الاعتماد على localStorage — قوليلي وقدر ساعدك فيه.
